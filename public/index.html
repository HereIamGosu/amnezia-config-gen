Генератор конфигов AmneziaWG для Cloudflare WARP

Создаёт готовый .conf для AWG 1.5 и AWG 2.0: WARP-ключи, AllowedIPs, DNS, CPS и endpoint-настройки.

Что вы получите

.conf-файл, WARP-ключи, DNS, маршруты, режим AWG 1.5 или AWG 2.0 и порт endpoint.

Когда использовать

Windows, Android, iOS, роутеры и split routing через пресеты AllowedIPs.

Ограничения

Работа зависит от доступности Cloudflare API, сети пользователя и поддержки режима клиентом AmneziaWG.

Конфигурацию можно сгенерировать для любых устройств. По умолчанию весь трафик идёт через туннель. Пресеты маршрутов и DNS можно задать в модальном окне по кнопке ⚙ рядом с кнопками генерации.

Для Windows — два отдельных .bat под версию клиента: 1.5 (Legacy) и 2.0; в каждом указан свой файл конфига в «Загрузках».

Инструкция по установке и настройке

  1. Скачивание и установка AmneziaWG:
    Перейдите по ссылке для скачивания и загрузите установочный файл для Windows. Установите по стандартному пути "C:\Program Files\...".
  2. Генерация конфигурационного файла:
    После установки нажмите «Сгенерировать AWG 1.5» (Legacy, файл AmneziaWarp.conf) или «Сгенерировать AWG 2.0» (файл AmneziaWarp-AWG2.conf) — в зависимости от версии клиента. Сохраните конфиг в папку «Загрузки»: bat планировщика ожидает именно эти имена файлов.
  3. Загрузка и запуск bat планировщика:
    Скачайте «Планировщик для 1.5» или «Планировщик для 2.0» — тот, что соответствует вашему конфигу. Запустите выбранный .bat от имени администратора один раз: будет создана задача в планировщике Windows при входе в систему. При смене версии клиента удалите старую задачу в Планировщике заданий или используйте другой .bat (имена задач внутри файлов различаются).

После этого AmneziaWG сможет автоматически подключаться к конфигу из «Загрузок» при входе в систему 🥸 Путь к amneziawg.exe в bat при необходимости поправьте под свою установку.

Дополнительная информация

🌐 Маршруты и AllowedIPs

По умолчанию весь трафик идёт через туннель (AllowedIPs = 0.0.0.0/0, ::/0). В настройках (кнопка ⚙) можно выбрать DNS-сервер и пресеты — тогда через туннель пойдут только нужные IPv4-подсети. IPv6 можно включить опционально, но это увеличивает число маршрутов в 2–3 раза и может вызывать сбои на роутерах и мобильных клиентах. Безопасный лимит — 1000 IPv4 CIDR.

IP-диапазоны можно заменить вручную. Актуальные списки — на opencck.org и rockblack.pro.

ℹ AWG 1.5 vs AWG 2.0

AWG 1.5 (Legacy)
  • Статические параметры обфускации (Jc, S1–S4, H1–H4)
  • Маскирует только начало соединения
  • Работает с Cloudflare WARP без сервера
  • Совместим со старыми клиентами
AWG 2.0
  • Динамические заголовки + случайный padding во всех пакетах
  • Имитирует QUIC/DNS на протяжении всей сессии
  • Сложнее для эвристического DPI-анализа
  • Требует клиент ≥ 4.8.12.9 (Amnezia)

⚠ Нужен ли сервер для AWG 2.0?

Для Cloudflare WARP — нет. Конфиги этого генератора работают с публичным сервером Cloudflare без каких-либо настроек на стороне сервера.

Для своего сервера — да. Самостоятельный AWG 2.0 требует обновлённого ядра/модуля amneziawg-linux-kernel-module на сервере и новых конфигов клиентов (старые конфиги AWG 1.5 становятся несовместимы). Подробнее — в официальной документации.
