# Ручная проверка `vpn://` в AmneziaVPN для iOS

Этот чек-лист подтверждает импорт конкретной сгенерированной ссылки в конкретной версии клиента. Он не гарантирует совместимость со всеми версиями AmneziaVPN.

## Подготовка

- [ ] Установлена актуальная на дату проверки версия AmneziaVPN из App Store.
- [ ] В приложении нет незавершённого экрана установки протокола или SSH-подключения.
- [ ] Открыт AmneziaWG Config Generator версии 2.4.1.
- [ ] Для проверки доступны буфер обмена iOS и стабильное интернет-соединение.

## Генерация и копирование

1. Выбрать режим `Legacy` или `AWG 2.0`.
2. Оставить стандартные настройки либо записать изменённые параметры в примечания ниже.
3. Выполнить генерацию и дождаться успешного результата.
4. Нажать кнопку копирования `vpn://`.
5. Убедиться, что строка в буфере начинается с `vpn://` и не была обрезана.

## Импорт в AmneziaVPN

1. Открыть AmneziaVPN на iOS.
2. Запустить импорт конфигурации из буфера обмена или вставить скопированную `vpn://` ссылку в доступный экран импорта.
3. Подтвердить импорт, не вводя SSH-доступ к серверу.
4. Проверить, что создан готовый профиль AmneziaWG.

## Ожидаемый результат

- [ ] Ссылка распознана как готовая конфигурация, а не как запрос установки протокола.
- [ ] Приложение не запрашивает SSH host, логин, пароль или ключ сервера.
- [ ] В импортированном профиле выбран AmneziaWG.
- [ ] Профиль содержит endpoint и может быть сохранён.
- [ ] После включения профиль переходит в состояние подключения.
- [ ] При доступной сети появляется handshake или передача трафика.

## Признаки ошибки

- Появляется экран установки протокола на сервер.
- Приложение запрашивает SSH-данные.
- Показывается ошибка разбора, декодирования или неизвестного контейнера.
- Импорт создаёт пустой профиль без endpoint.
- Ссылка отклоняется сразу после вставки.
- Профиль сохраняется, но не запускается; этот случай нужно отделять от сетевой недоступности endpoint.
